# commitlint-config-zx

用来检查 git 提交信息的规则

## 提交格式

{类型}:{空格}{描述}

```js
// 所有类型
[
  // 源码性修改
  'feat', // 增加新功能
  'dev', // 开发阶段的代码变更
  'debug', // 调试代码
  'fix', // 修复bug
  'pref', // 性能优化
  'refactor', // 重构代码
  'demand', // 业务需求变更
  'user', // 用户体验
  'style', // 不影响代码逻辑只影响代码外观的代码变更 例如：代码风格变更，空格，回车，注释，变量命名等
  // 非源码性修改
  'depend', // 项目依赖变更
  'docs', // 文档说明相关变更
  'test', // 测试功能相关变更
  'build', // 打包
  'chore', // 非源码性的其他杂项修改
]
```
